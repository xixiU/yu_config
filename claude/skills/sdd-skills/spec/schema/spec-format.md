# /spec 产出物格式规范

## 概述

/spec 产出多份文档，通过 ID 系统互相引用，驱动后续 /develop、/review、/testing。

| 文档 | 回答 | 路径 |
|------|------|------|
| spec.md | 做什么 | .ai-workspace/spec.md |
| contracts.md | 接口契约 | .ai-workspace/contracts.md |
| testcases.md | 怎么验 | .ai-workspace/testcases.md |
| design.md | 怎么做 | .ai-workspace/design.md |
| tasks.md | 具体步骤 | .ai-workspace/tasks.md |

## ID 编号规则

| 类型 | 前缀 | 示例 | 定义位置 | 引用位置 |
|------|------|------|---------|---------|
| 需求目标 | G- | G-001 | spec.md §1.2 | 全部文档 |
| 非功能需求 | NFR- | NFR-001 | spec.md §1.3 | design.md, tasks.md |
| 验收标准 | AC- | AC-001 | testcases.md §1 | testcases.md §2 |
| 决策点 | DEC- | DEC-001 | design.md §2 | spec.md（追溯矩阵） |
| 接口契约 | API- | API-001 | contracts.md | tasks.md, design.md |
| 测试用例 | TC- | TC-001 | testcases.md §2 | tasks.md |
| 风险 | R- | R-001 | spec.md §4 | design.md |
| 任务 | TASK- | TASK-001 | tasks.md §2 | spec.md（追溯矩阵） |

所有 ID 在文档间唯一，通过 _ref 字段互相引用。

```markdown
# 验收标准与测试用例

> 关联: spec.md v1.0, contracts.md

## 1. 验收标准

| ID | 关联需求 | Given | When | Then |
|----|---------|-------|------|------|
| AC-001 | G-001 | 数学科组长登录 | 访问数学科目阅卷 | 正常显示 |
| AC-002 | G-001 | 数学科组长登录 | 访问英语科目阅卷 | 拒绝访问 |
| AC-003 | G-001 | 历史数据 subjectCode=null | 访问任意科目 | 视为全科权限 |
| AC-004 | NFR-001 | 并发 100 请求 | 调用权限接口 | P99 < 200ms |

## 2. 测试用例

| ID | AC | 接口 | 类型 | 级别 | 标题 | 前置 | 输入 | 期望 |
|----|-----|------|------|------|------|------|------|------|
| TC-001 | AC-001 | API-001 | unit | P1 | 本科目返回true | math科组长 | subjectCode=math | true |
| TC-002 | AC-002 | API-001 | unit | P1 | 非本科目返回false | math科组长 | subjectCode=english | false |
| TC-003 | AC-003 | API-001 | unit | P2 | 旧数据兼容 | subjectCode=null | subjectCode=math | true |
| TC-004 | AC-004 | - | perf | P2 | 性能不退化 | 100并发 | 批量请求 | P99<200ms |
```

字段说明：
- AC: 关联的验收标准 AC-xxx
- 类型: unit / integration / e2e / perf / security
- 级别: P1（核心路径）/ P2（重要分支）/ P3（边界场景）

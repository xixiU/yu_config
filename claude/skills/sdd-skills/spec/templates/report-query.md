# 报表查询 Spec 模板

适用场景：新增报表页面、数据查询接口、导出功能等。

## 第一章：需求

### G-001: {报表名称}查询
- 描述：{用户需要查询什么数据，支持哪些筛选条件}
- 验收标准：
  - [ ] 支持按{条件1}、{条件2}筛选
  - [ ] 分页查询，默认每页20条
  - [ ] 支持导出 Excel

### G-002: {报表名称}导出
- 描述：将查询结果导出为 Excel 文件
- 验收标准：
  - [ ] 导出字段与页面展示一致
  - [ ] 大数据量异步导出（超过5000条）

### API-001: 分页查询接口
```
POST /api/v1/{模块}/{资源}/query
Request:
  - pageNum: int (默认1)
  - pageSize: int (默认20, 最大100)
  - {筛选条件字段}
Response:
  - code: int
  - data:
    - total: int
    - list: Array<{资源}VO>
```

### API-002: 导出接口
```
POST /api/v1/{模块}/{资源}/export
Request:
  - {筛选条件字段，同查询}
Response:
  - 同步模式: 文件流 (Content-Type: application/octet-stream)
  - 异步模式: { code, data: { taskId, downloadUrl } }
```

## 第三章：测试用例

### TC-001: 无筛选条件查询
- 前置：数据库有测试数据
- 步骤：不填任何条件，点击查询
- 预期：返回全量数据第一页，分页正确

### TC-002: 单条件筛选
- 前置：同上
- 步骤：填写{条件1}，点击查询
- 预期：结果只包含匹配数据

### TC-003: 多条件组合筛选
- 步骤：同时填写{条件1}+{条件2}
- 预期：结果满足所有条件（AND 逻辑）

### TC-004: 空结果查询
- 步骤：填写不存在的条件值
- 预期：显示"暂无数据"，total=0

### TC-005: 分页翻页
- 步骤：查询后点击第2页
- 预期：数据正确切换，页码高亮

### TC-006: 导出 - 正常导出
- 步骤：查询有数据 → 点击导出
- 预期：Excel 下载成功，内容与页面一致

### TC-007: 导出 - 空数据导出
- 步骤：查询无数据 → 点击导出
- 预期：提示"无数据可导出"或导出空表头

### TC-008: 大数据量查询性能
- 步骤：不加筛选条件查询（数据量>1万）
- 预期：3秒内返回结果
